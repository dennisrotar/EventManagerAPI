# Event Manager API

Сервис для управления мероприятиями. Базовый каркас, реализованный на ASP.NET Core Web API.

## Технологический стек

- C# / .NET 10
- ASP.NET Core Web API
- In-memory хранилище
- Swagger (OpenAPI)
- xUnit (юнит-тестирование)

## Запуск проекта

1. Клонируйте репозиторий:
    ```bash
    git clone https://github.com/dennisrotar/EventManagerAPI.git EventManagerAPI
    ```

2. Перейдите в репозиторий
    ```bash
    cd EventManagerAPI
    ```

3. Соберите проект:
    ```bash
    dotnet build
    ```

4. Запустите проект:
    ```bash
    dotnet run
    ```

5. Откройте браузер и перейдите по адресу: `http://localhost:<порт>/swagger` (порт будет указан в консоли после запуска).

## Запуск тестов

Для запуска юнит-тестов выполните команду в корне решения:
    ```bash
    dotnet test
    ```

## Документация API

Базовый путь: `/events`

|Метод|Эндпоинт|Описание|Успешный статус|
|---|---|---|---|
|`GET`|`/events`|Получить список всех мероприятий|200 OK|
|`GET`|`/events/{id}`|Получить мероприятие по ID|200 OK|
|`POST`|`/events`|Создать новое мероприятие|201 Created|
|`PUT`|`/events/{id}`|Обновить мероприятие целиком|204 No Content|
|`DELETE`|`/events/{id}`|Удалить мероприятие|204 No Content|

## Фильтрация и пагинация (GET /events)

Эндпоинт `GET /events` принимает query-параметры:

- title (string) — поиск по названию (частичное совпадение, без учета регистра).
- from (DateTime) — события, начинающиеся не раньше этой даты.
- to (DateTime) — события, заканчивающиеся не позже этой даты.
- page (int, по умолчанию 1) — номер страницы.
- pageSize (int, по умолчанию 10) — размер страницы.

### Пример запроса:
GET /events?title=концерт&from=2024-01-01T00:00:00Z&page=2&pageSize=5

### Пример ответа пагинации:

```json
{
  "totalCount": 12,
  "page": 2,
  "pageSize": 5,
  "items": [ ... ]
}
```

## Обработка ошибок

Все ошибки возвращаются в едином формате **Problem Details (RFC 7807)**.

**Пример ответа при ошибке 404 (Не найдено) или 500 (Внутренняя ошибка):**

```json
{
"title": "Not Found",
"status": 404,
"detail": "Мероприятие с ID ... не найдено",
"instance": "/events/..."
}
```
Ошибки валидации (например, пустой Title) автоматически возвращают статус 400 Bad Request.
